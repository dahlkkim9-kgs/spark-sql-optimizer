# Spark SQL 优化工具 - 使用说明

## 📦 安装说明

### 方式一：安装程序（推荐）

1. 双击运行 `Spark SQL优化工具 Setup 1.0.0.exe`
2. 选择安装目录（默认：`C:\Users\用户名\AppData\Local\Programs\spark-sql-optimizer`）
3. 等待安装完成
4. 双击桌面图标启动应用

### 方式二：免安装版

1. 解压 `win-unpacked` 文件夹
2. 双击 `Spark SQL优化工具.exe` 启动应用

## 🚀 快速开始

### 界面说明

```
┌─────────────────────────────────────────────────────────┐
│  ⚡ Spark SQL 优化工具                                    │
│  离线静态分析 · 智能优化建议                               │
├─────────────────────────────────────────────────────────┤
│  SQL编辑器                          [📁 上传][清空][分析SQL]  │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 1 │ SELECT * FROM users CROSS JOIN orders      │  │
│  │ 2 │ WHERE name LIKE '%john%'                   │  │
│  └───────────────────────────────────────────────────┘  │
├─────────────────────────────────────────────────────────┤
│  分析结果                    [高: 2] [中: 1] [低: 0]      │
│  ┌───────────────────────────────────────────────────┐  │
│  │ [高] SELECT_STAR     第 1 行                     │  │
│  │ 避免使用 SELECT *，只查询需要的字段                │  │
│  │ 建议：列出具体需要的字段名                         │  │
│  │ 代码：SELECT * FROM users                        │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 操作步骤

1. **输入 SQL**
   - 直接在编辑器中输入
   - 或点击"上传文件"选择 .sql/.txt 文件

2. **分析**
   - 点击"分析SQL"按钮
   - 等待分析完成（通常 < 1 秒）

3. **查看结果**
   - 高优先级问题（红色）
   - 中优先级问题（橙色）
   - 低优先级问题（绿色）

4. **优化建议**
   - 查看每个问题的优化建议
   - 复制优化后的 SQL

## 🔍 检测规则

### 高优先级 (HIGH)

| 规则 | 说明 |
|-----|------|
| SELECT_STAR | 避免使用 SELECT * |
| LIKE_LEADING_WILDCARD | LIKE 以通配符开头 |
| CROSS_JOIN | 笛卡尔积风险 |
| JOIN_NO_CONDITION | JOIN 缺少 ON 条件 |

### 中优先级 (MEDIUM)

| 规则 | 说明 |
|-----|------|
| OR_IN_WHERE | 多个 OR 条件 |
| HAVING_WHERE | HAVING 条件可下推 |
| COUNT_DISTINCT_LARGE | COUNT DISTINCT 可能数据倾斜 |
| SUBQUERY_DEEP | 嵌套子查询 |

### 低优先级 (LOW)

| 规则 | 说明 |
|-----|------|
| DISTINCT_ORDER_BY | DISTINCT + ORDER BY 性能开销 |
| HARDCODED_DATE | 硬编码日期 |
| LIMIT_MISSING | ORDER BY 无 LIMIT |

## 📝 示例 SQL

```sql
-- 测试用例：包含多个问题
SELECT *
FROM users
CROSS JOIN orders
WHERE name LIKE '%john%'
  OR status = 'A'
  OR status = 'B'
ORDER BY create_time;
```

**检测结果：**
- ❌ SELECT * (高)
- ❌ CROSS JOIN (高)
- ❌ LIKE 以 % 开头 (高)
- ⚠️ 多个 OR 条件 (中)
- ⚠️ ORDER BY 无 LIMIT (低)

## 🐛 常见问题

### Q: 应用无法启动？
A: 检查是否被杀毒软件阻止，添加到白名单后重试。

### Q: 分析后无反应？
A: 查看控制台日志（F12），检查后端是否启动。

### Q: 想要添加自定义规则？
A: 联系开发者或等待后续版本。
