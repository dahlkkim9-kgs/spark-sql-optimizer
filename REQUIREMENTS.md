# Spark SQL 优化工具 - 需求说明书

## 1. 项目概述

### 1.1 项目背景
企业内部 Spark SQL 脚本开发过程中，缺乏有效的代码质量检查工具，导致性能问题频发。需要一款离线可用的静态分析工具，帮助开发人员及时发现 SQL 代码中的潜在问题。

### 1.2 项目目标
开发一款 **离线可用** 的 Spark SQL 静态分析和优化工具，通过静态代码分析发现性能问题并提供优化建议。

### 1.3 目标用户
- Spark SQL 开发人员
- 数据工程师
- 大数据平台维护人员

### 2.2 分析规则

#### 高优先级规则 (HIGH)
| 规则名称 | 问题描述 | 建议 |
|---------|---------|------|
| SELECT_STAR | 避免使用 SELECT * | 列出具体字段名 |
| LIKE_LEADING_WILDCARD | LIKE 以通配符开头无法使用索引 | 考虑全文索引 |
| CROSS_JOIN | 笛卡尔积风险 | 检查 JOIN 条件 |
| JOIN_NO_CONDITION | JOIN 缺少 ON 条件 | 添加 JOIN 条件 |

#### 中优先级规则 (MEDIUM)
| 规则名称 | 问题描述 | 建议 |
|---------|---------|------|
| OR_IN_WHERE | 多个 OR 条件可能全表扫描 | 使用 IN 或 UNION ALL |
| HAVING_WHERE | HAVING 条件可下推到 WHERE | 将条件移到 WHERE |
| COUNT_DISTINCT_LARGE | COUNT(DISTINCT) 可能数据倾斜 | 使用 approx_count_distinct |
| SUBQUERY_DEEP | 嵌套子查询可读性差 | 使用 CTE（WITH 子句） |

#### 低优先级规则 (LOW)
| 规则名称 | 问题描述 | 建议 |
|---------|---------|------|
| DISTINCT_ORDER_BY | DISTINCT + ORDER BY 性能开销大 | 考虑是否需要排序 |
| HARDCODED_DATE | 硬编码日期灵活性差 | 使用参数变量 |
| LIMIT_MISSING | ORDER BY 无 LIMIT 可能内存溢出 | 添加 LIMIT |

## 3. 非功能需求

### 3.1 性能要求
- 单个 SQL 文件分析时间 < 2 秒
- 支持 100MB 以内的 SQL 文件
- 支持同时分析 100+ 个文件

### 3.2 兼容性要求
- **操作系统**: 统信UOS (Debian系)、Windows 10+、macOS
- **Python**: 3.9+ (与现有 Python 2.7 共存)
- **部署方式**: 完全离线，无需互联网连接

### 3.3 易用性要求
- 界面简洁直观，学习成本低
- 支持拖拽文件上传
- 一键安装部署

## 4. 技术架构

### 4.1 技术选型

| 层级 | 技术栈 | 说明 |
|-----|-------|------|
| **前端** | React 18 + Monaco Editor | 现代化 UI 框架 |
| **后端** | Python 3.9+ + FastAPI | 轻量级 REST API |
| **桌面** | Electron | 跨平台桌面应用 |
| **SQL解析** | SqlGlot | 纯 Python SQL 解析器 |

### 4.2 项目结构
```
spark-sql-optimizer/
├── frontend/          # React 前端
├── backend/           # Python 后端
│   ├── api/          # FastAPI 接口
│   └── core/         # 分析器核心
├── electron/          # Electron 桌面应用
├── resources/         # 规则库文档
└── dist/             # 打包输出
```

## 5. 部署方案

### 5.1 统信UOS 离线部署

1. 在联网机器下载依赖包
2. 传输到离线机器
3. Python 3 与 Python 2.7 共存：
   - Python 2.7: `python` 命令
   - Python 3: `python3` 命令

### 5.2 Windows 本地开发

```bash
# 1. 安装依赖
start.bat          # Windows 一键安装脚本

# 2. 启动测试
test.bat           # 同时启动前后端

# 3. 打包发布
npm run build:win  # 生成 .exe 安装包
```
