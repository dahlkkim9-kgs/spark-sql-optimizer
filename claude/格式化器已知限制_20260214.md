# SQL格式化器已知限制

> Last Updated: 2026-02-27

## V4 - 支持注释保留 (formatter_v4.py)

V4版本格式化器已解决V3的注释保留问题。

### V4功能特性

| 功能 | 状态 | 说明 |
|-----|------|------|
| 行注释保留 (`--`) | 支持 | 字段级、表级、CASE语句内的注释都能保留 |
| 块注释保留 (`/* */`) | 支持 | 多行块注释可正确保留 |
| 注释位置保持 | 支持 | 注释会保留在相关元素后面 |
| 中文注释 | 支持 | 完整支持中文注释 |

### V4使用方法

```python
from core.formatter_v4 import format_sql_v4

sql = """SELECT id -- 主键
     , name -- 名称
FROM table1 -- 主表
;"""

result = format_sql_v4(sql)
# 注释将被保留在格式化后的SQL中
```

### V4架构

V4是对V3的包装增强:
1. `CommentPreserver` - 提取和重新插入注释
2. `formatter_v3` - 执行实际的格式化逻辑
3. 三个阶段: 提取注释 -> 替换为占位符 -> 格式化 -> 重新插入注释

## V3 - 原始版本 (formatter_v3.py)

以下限制仅适用于V3版本。V4版本已解决这些问题。

### V3已知限制

| 限制项 | 说明 | 影响程度 |
|-------|------|---------|
| **注释保留** | V3会丢失所有注释 | 高 |
| 字段名/表名大小写 | 部分场景可能不一致 | 低 |
| 复杂子查询 | 嵌套子查询格式化可能不理想 | 中 |

### V3注释丢失示例

```sql
-- V3输入
SELECT id -- 主键
     , name -- 名称
FROM table1;

-- V3输出 (注释丢失)
SELECT
       id
     , name
FROM
      table1
;
```

### 何时使用V3 vs V4

- **使用V4**: 需要保留注释的生产环境SQL
- **使用V3**: 不需要注释的简单SQL或特殊场景
