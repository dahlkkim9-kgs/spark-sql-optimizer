## 三、已完成工作

### 3.1 问题分析

**原始问题**: 格式化嵌套 CASE WHEN 时，无论原始 SQL 是否有括号，都会强制添加括号

**问题代码位置**: `formatter_v4.py` 第 435-446 行（THEN 子句）和第 458-474 行（ELSE 子句）

### 3.2 代码修改

**修改文件**: `spark-sql-optimizer/backend/core/formatter_v4.py`

**修改点 1** - THEN 子句嵌套 CASE（第 435-452 行）:
```python
# 修改前：总是添加括号
lines.append(f'{inner_indent}THEN (')
...
lines.append(f'{inner_indent}    )')

# 修改后：根据原始是否有括号决定
if has_paren_wrapper:
    # 原始有括号，保留括号
    lines.append(f'{inner_indent}THEN (')
    for nested_line in nested_formatted.split('\n'):
        lines.append(nested_line)
    lines.append(f'{inner_indent}    )')
else:
    # 原始没有括号，不加括号
    lines.append(f'{inner_indent}THEN')
    for nested_line in nested_formatted.split('\n'):
        lines.append(nested_line)
```

**修改点 2** - ELSE 子句嵌套 CASE（第 466-482 行）:
- 同样的逻辑应用到 ELSE 部分

### 4.1 WHERE 子句注释优化 ✅ 已完成

**日期**: 2026-03-02

**问题**: WHERE 子句中括号包裹的 OR 组合条件，当括号后有注释时，注释被错误地合并到一行。

**原因分析**:
- `_format_condition_with_ors` 函数检查 `condition.endswith(')')` 来判断是否是括号组
- 当括号后有注释（如 `__COMMENT_2__`）时，条件不满足，导致整个内容被当作单行处理

**修复方案**:
修改 `_format_condition_with_ors` 函数：
1. 先查找匹配的右括号位置
2. 提取括号后的注释（如有）
3. 单独处理括号内的 OR 条件分割
4. 将括号后注释附加到闭合括号后

**修改文件**: `formatter_v4.py` 第 750-816 行

**测试文件**: `tests/test_where_comments.py`

**测试用例**:
- `test_where_simple_comments` - 简单 WHERE 条件注释
- `test_where_or_group_with_comments` - 括号 OR 组注释
- `test_where_multiple_conditions_with_comments` - 多条件注释
- `test_where_nested_parens_with_comments` - 嵌套括号注释
